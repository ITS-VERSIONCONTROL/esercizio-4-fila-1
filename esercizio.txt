Per questo esercizio utilizzeremo i seguenti branch:
	develop: branch di sviluppo da cui staccare le nuove features
	main: branch di rilascio da cui staccare i nuovi rilasci


### STEP 1 ###

- scegliere una feature da implementare

- staccare un feature branch da develop
	partendo dal branch di develop, creare un feature branch, spostarsi nel nuovo branch
	(git branch, git switch)

- nel feature branch implementare la feature e fare la push del feature branch su origin
	controllare che il proprio feature branch sia consistente e funzioni correttamente
	(git push -u origin)

- riallineare il merge di develop
	spostarsi sul branch di develop, e riallinearsi con origin (git fetch, git pull)
	riallineare il feature branch (git merge)
	risolvere eventuali conflitti rispettando l'ordine del menu
	verificare che tutto funzioni correttamente
	effettuare la commit delle modifiche effettuate
	effettuare il push di develop su origin (git push)
